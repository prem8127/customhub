# CustomHub Backend Migration

This backend replaces the previous Flask API with a Node.js, Express, TypeScript, PostgreSQL, and Prisma architecture while preserving the existing frontend API contract.

## Architecture

- `src/app.ts`: Express app composition, security middleware, CORS, rate limits, route registration.
- `src/server.ts`: server boot and graceful shutdown.
- `src/config/env.ts`: centralized environment validation with Zod.
- `src/db/prisma.ts`: shared Prisma client.
- `src/middleware`: auth, request validation, not found, and error handling.
- `src/modules/auth`: email/password auth, Google auth, JWT access tokens, refresh token rotation.
- `src/modules/products`: product listing and detail APIs.
- `src/modules/orders`: checkout order creation and order history.
- `src/modules/payments`: Razorpay order creation and signature verification.
- `src/modules/customization`: Cloudinary upload and preview endpoints.
- `src/data/products.ts`: seed catalog equivalent to the original Flask seed data.
- `prisma/schema.prisma`: PostgreSQL schema and relations.
- `prisma/migrations`: production migration files.

## API Compatibility

The following legacy paths are preserved:

- `POST /api/register`
- `POST /api/login`
- `POST /api/google-auth`
- `GET /api/products`
- `GET /api/product/:productId`
- `POST /api/upload-design`
- `POST /api/preview`
- `POST /api/order`
- `GET /api/orders`
- `POST /api/create-order`
- `POST /api/verify-payment`
- `GET /api/health`

Auth responses still include `token` for the existing frontend. New clients can use `accessToken` and `refreshToken`.

## Environment

Required:

```bash
NODE_ENV=development
PORT=5001
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/customhub
FRONTEND_ORIGIN=http://localhost:3000
JWT_ACCESS_SECRET=change-this-access-secret
JWT_REFRESH_SECRET=change-this-refresh-secret
COOKIE_SECRET=change-this-cookie-secret
```

## Local Migration Steps

1. Start PostgreSQL:

```bash
docker compose up -d postgres
```

2. Install backend dependencies:

```bash
cd backend
npm install
```

3. Create `backend/.env` from `backend/.env.example` and set secure secrets.

4. Apply migrations and seed products:

```bash
npm run prisma:generate
npm run prisma:migrate
npm run prisma:seed
```

5. Start the backend:

```bash
npm run dev
```

6. Start the frontend:

```bash
cd ../frontend
npm install
npm run dev
```

## SQLite To PostgreSQL Data Strategy

The product catalog is deterministic and is reseeded from `src/data/products.ts`.

For existing users and orders in the old SQLite database:

1. Export `users` and `orders` from `backend/instance/customhub-dev.sqlite3`.
2. Transform column names from snake_case to the Prisma schema fields.
3. Insert into PostgreSQL with Prisma Client or `psql`.
4. Keep `password_hash` values if they were generated by bcrypt. They remain compatible.
5. Do not migrate old JWTs. Users should sign in again and receive Node-issued access and refresh tokens.

## Deployment

### Render or Railway

- Build command: `npm ci && npx prisma generate && npm run build`
- Start command: `npx prisma migrate deploy && npm run start`
- Root directory: `backend`
- Add PostgreSQL and set `DATABASE_URL`.
- Set `FRONTEND_ORIGIN` to the Vercel frontend URL.
- Set strong JWT and cookie secrets.

### Vercel Frontend

- Root directory: `frontend`
- Set `NEXT_PUBLIC_API_URL=https://your-backend-host/api`
- Keep the existing Next.js build command.

### Docker

The root `docker-compose.yml` runs PostgreSQL and the backend. The frontend remains deployable independently on Vercel or can be run locally with `npm run dev`.
